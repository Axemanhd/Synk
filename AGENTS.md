## Reports
| File | Purpose |
|------|---------|
| `AUDIT.md` | Full codebase audit log — all bugs, workflow issues, security concerns |
| `PROGRESS.md` | Completed features, roadmap, and audit issue tracking |

## Session End
When the user says "we are done" or "that's all for now" or similar:
1. Run a full codebase audit — check every issue in `AUDIT.md` against the current source code
2. Update `AUDIT.md` and `PROGRESS.md` with the findings (version, date, resolved/still-open counts)
3. Report the audit results in a summary table
4. Confirm I'm done
